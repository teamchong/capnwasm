Live chat — three helpers in 30 lines

A small createClient + subscribeQuery + optimistic demo. Open this page in two tabs; messages stream live across. Source: web/src/chat/main.ts.

What you're seeing

On submit, the message appears immediately (optimistic apply). If the server confirms, the local entry is replaced by the server's authoritative copy when the next stream chunk arrives. If the call rejects, the local entry turns red and shows the error.

The whole client, more or less

import { createClient, subscribeQuery, optimistic } from "capnwasm/client";

const { cap } = await createClient("/chat");

const sub = subscribeQuery(cap, IFC, METHOD_WATCH, EMPTY_PARAMS);
for await (const chunk of sub.updates) {
  render(JSON.parse(textDecoder.decode(chunk)));
}

await optimistic({
  apply: () => renderPending(text),
  send:  () => cap.call(IFC, METHOD_POST, encodeJson({ author, text })).promise,
  revert: () => markFailed(text),
});

The wire bytes here are JSON for clarity — a real deployment would use a Cap'n Proto schema for the message struct. The helpers stay the same regardless of wire format; only the encode/decode lines change. See the playground for binary-wire perf comparisons.
