capnwasm

Real Cap'n Proto C++ in the browser. Typed RPC, binary wire, 39 KB gzip.

One typed-client toolchain for any backend — write a .capnp schema, get a fast typed JS client back. npx capnwasm gen user.capnp emits a reader/builder that runs against real upstream Cap'n Proto C++ statically compiled to wasm.

vs capnweb & REST/JSON

RPC burst, 200 concurrent calls (browser→Node WS) ~4–6× faster than capnweb
RPC 64 KB binary echo (browser→Node WS) ~1.3–1.7× faster, 25% less wire
Wire bytes for 4 KB binary blob (gzip) 2.4× smaller than JSON / capnweb
In-browser parallel HTTP fetch (small records) essentially tied — localhost network dominates
Bundle (JS + wasm, gzip) 39 KB — 1.85× larger than capnweb
Cold start (Node, fresh process, total) ~1 ms — tied with capnweb
Cold start (browser, fresh tab w/ empty cache) ~20 ms — wasm fetch + compile, vs capnweb's 1–3 ms JS parse
